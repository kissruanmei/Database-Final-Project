# 需求分析

## 一、项目背景

校园保卫处交通管理部门需要管理进出校园的机动车，目前采用人工登记、纸质记录的方式，存在以下问题：
- 车辆注册信息分散，难以统一查询和管理
- 违规记录靠纸质台账，容易遗漏丢失
- 车主办事（注册、处理违规）需要线下跑窗口，效率低
- 缺乏统计分析手段，无法掌握违规趋势

本系统旨在建设一套**数字化校园机动车综合管理平台**，实现车辆注册、违规记录、处罚处理与数据查询的全流程线上化。

---

## 二、用户角色

| 角色 | 说明 | 主要操作 |
|------|------|----------|
| **管理员**（保卫处工作人员） | 系统的主要操作者，负责日常管理 | 审批注册、录入违规、处理处罚、查询统计 |
| **车主** | 校园内机动车的拥有者（教职工/学生/临时访客） | 提交注册申请、查询违规记录、处理处罚 |

---

## 三、功能需求

### 功能 1：车辆权限注册管理

- 车主提交注册申请，填写个人信息和车辆信息
- 管理员审核申请，审批通过后发放通行证
- 通行证包含：权限类型、有效期、允许进入的区域
- 支持通行证续期、注销

### 功能 2：违规记录管理

- 管理员录入违规信息（车牌号、违规类型、地点、时间）
- 系统自动关联车辆和车主信息
- 违规类型包括：违停、超速、无证入校、占用消防通道等
- 每次违规扣减相应分数

### 功能 3：处罚处理

- 车主查询自己的违规记录和待处理处罚
- 管理员执行处罚操作（罚款、警告、禁止入校等）
- 车主累计违规分数达到阈值时，自动触发处罚（如暂停通行证）
- 支持处罚申诉流程

### 功能 4：数据查询与统计

- 按车牌号/车主姓名查询违规历史
- 按时间段统计违规数量
- 按违规类型/地点统计违规分布
- 按学院统计违规情况
- 导出查询结果

## 六、业务规则

1. **注册规则**：一个车主可以注册多辆车，一辆车只能对应一个车主
2. **通行证规则**：每辆车同时只能持有一张有效通行证
3. **违规积分规则**：每次违规根据类型扣减 1-6 分，累计扣分达到 12 分暂停通行证
4. **处罚规则**：违停罚款50元、超速罚款100元、无证入校罚款200元、占用消防通道罚款500元
5. **申诉规则**：车主可在处罚生效后7日内提出申诉，由管理员审核
6. **通行证续期**：到期前可申请续期，过期后自动变为"过期"状态

## 七、非功能需求

1. **数据安全**：管理员需登录验证，密码加密存储
2. **数据完整性**：车牌号唯一、身份证号唯一、级联删除保护
3. **易用性**：CLI 界面菜单清晰，操作提示友好
4. **性能**：单条查询响应时间 < 1秒
