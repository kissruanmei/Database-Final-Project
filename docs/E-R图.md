## 二、联系（Relationship）

| 联系名 | 实体A | 实体B | 对应关系 | 说明 |
|--------|-------|-------|----------|------|
| 拥有 | 车主 | 车辆 | 1 : N | 一个车主拥有多辆车，一辆车属于一个车主 |
| 持有 | 车辆 | 通行证 | 1 : 1 | 一辆车同时持有一张有效通行证 |
| 产生 | 车辆 | 违规记录 | 1 : N | 一辆车可产生多条违规记录 |
| 对应 | 违规记录 | 处罚记录 | 1 : 1 | 一条违规记录对应一条处罚记录 |
| 审批 | 管理员 | 通行证 | 1 : N | 一个管理员可审批多张通行证 |
| 录入 | 管理员 | 违规记录 | 1 : N | 一个管理员可录入多条违规记录 |
| 处理 | 管理员 | 处罚记录 | 1 : N | 一个管理员可处理多条处罚记录 |

## 三、E-R 图（文字版）

```
                         ┌──────────┐
                         │  管理员   │
                         │ (Admin)  │
                         └────┬─────┘
                    审批/录入/处理│
              ┌────────────────┼────────────────┐
              │                │                │
              ▼                ▼                ▼
        ┌──────────┐    ┌──────────┐    ┌──────────┐
        │  通行证   │    │ 违规记录  │    │ 处罚记录  │
        │ (Permit) │    │(Violation)│    │(Penalty) │
        └─────┬────┘    └─────┬────┘    └─────┬────┘
              │ 1:1           │ 1:N           │ 1:1
              │               │               │
              ▼               ▼               │
        ┌──────────┐    ┌──────────┐    违规记录↔处罚记录
        │  车辆     │◄───│  车辆     │
        │(Vehicle) │1:N │(Vehicle) │
        └─────┬────┘    └──────────┘
              │ 1:N
              ▼
        ┌──────────┐
        │  车主     │
        │ (Owner)  │
        └──────────┘


  完整关系链：

  管理员 ──(审批 1:N)──→ 通行证 ←──(持有 1:1)── 车辆 ←──(拥有 N:1)── 车主
                          │
  管理员 ──(录入 1:N)──→ 违规记录 ←──(产生 1:N)── 车辆
                          │
  管理员 ──(处理 1:N)──→ 处罚记录 ←──(对应 1:1)── 违规记录
```

## 四、E-R 图绘制说明

## 五、实体-联系总结

本系统共 **6 个实体**、**7 个联系**：

| # | 实体 | 主键 | 联系数 |
|---|------|------|--------|
| 1 | 车主 Owner | owner_id | 1（拥有多辆车） |
| 2 | 车辆 Vehicle | plate_number | 3（属于车主、持有通行证、产生违规） |
| 3 | 通行证 Permit | permit_id | 2（被车辆持有、被管理员审批） |
| 4 | 违规记录 Violation | violation_id | 3（由车辆产生、由管理员录入、对应处罚） |
| 5 | 处罚记录 Penalty | penalty_id | 2（对应违规、被管理员处理） |
| 6 | 管理员 Admin | admin_id | 3（审批通行证、录入违规、处理处罚） |
