# SQL 脚本执行指南

## 执行顺序

请按照以下顺序在 SQL Server Management Studio (SSMS) 中执行脚本：

### 步骤 1：创建数据库
```sql
-- 在 master 数据库中执行
CREATE DATABASE CampusVehicleDB;
GO
```

### 步骤 2：执行建表脚本
1. 打开 SSMS，连接到 SQL Server
2. 新建查询，选择 `CampusVehicleDB` 数据库
3. 打开并执行 `01_create_tables.sql`

### 步骤 3：插入测试数据
1. 确保当前数据库为 `CampusVehicleDB`
2. 打开并执行 `02_insert_data.sql`

### 步骤 4：创建视图
1. 打开并执行 `03_views.sql`
2. 执行后可通过以下查询验证：
   ```sql
   SELECT * FROM vw_OwnerViolationHistory;
   SELECT * FROM vw_MonthlyViolationStats;
   ```

### 步骤 5：创建存储过程
1. 打开并执行 `04_procedures.sql`
2. 执行后可通过以下方式测试：
   ```sql
   -- 测试录入违规记录
   EXEC sp_AddViolation
       @plate_number = '京A10001',
       @violation_type = N'违停',
       @violation_time = '2026-06-05 10:00:00',
       @violation_location = N'教学楼前',
       @violation_desc = N'占用人行道',
       @points_deducted = 2,
       @admin_id = 1;

   -- 测试查询车主违规历史
   EXEC sp_GetOwnerViolations @owner_id = 1;
   ```

### 步骤 6：创建触发器
1. 打开并执行 `05_triggers.sql`
2. 触发器会自动生效，无需手动调用

## 常见问题

### 1. 中文乱码问题
- 确保在 SSMS 中使用 `Unicode` 编码打开 .sql 文件
- 在连接属性中设置正确的字符集

### 2. 外键约束错误
- 如果遇到外键约束错误，请检查表的创建顺序
- 确保按照 Admin → Owner → Vehicle → Permit → Violation → Penalty 的顺序创建

### 3. 数据已存在错误
- 脚本开头有清理语句，会先删除旧表
- 如果只想插入数据，请注释掉清理部分

## 使用 Python 执行

可以使用 `run_sql.py` 脚本批量执行：
```bash
python sql/run_sql.py
```

## 验证执行结果

执行完成后，可通过以下查询验证：
```sql
-- 查看所有表
SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_TYPE = 'BASE TABLE';

-- 查看所有视图
SELECT TABLE_NAME FROM INFORMATION_SCHEMA.VIEWS;

-- 查看所有存储过程
SELECT ROUTINE_NAME FROM INFORMATION_SCHEMA.ROUTINES WHERE ROUTINE_TYPE = 'PROCEDURE';

-- 查看所有触发器
SELECT name FROM sys.triggers;
```
